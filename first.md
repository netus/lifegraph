项目名称：LifeGraph

产品本质：
一个把“人生经历”结构化为数据节点，并形成时间 + 地点 + 人物关系网络的系统

--------------------------------

一、核心定义（不会变的底层逻辑）

一个节点 = 一次真实经历

节点结构 = 三维坐标 + 内容
- 时间（when）
- 地点（where）
- 参与者（who）
- 事件（what）

本质：
把人生变成“可记录、可检索、可连接”的数据

--------------------------------

二、核心能力（产品必须具备）

1）极简输入
用户只需：
- 一句话
或
- 语音 / 图片 / 视频

系统自动生成节点

--------------------------------

2）节点结构化
系统自动解析：
- 时间
- 地点
- 人物
- 事件内容
- 标签 / 情绪（扩展）

--------------------------------

3）人生记录系统
用户可以：
- 查看时间线（Timeline）
- 按时间回看人生
- 编辑 / 删除节点

--------------------------------

4）检索能力
可以按：
- 时间
- 地点
- 人物
- 关键词

查找人生经历

--------------------------------

5）社交关系（核心差异化）

一个节点可以属于多个用户

机制：
- A 创建节点
- 标记 B
- B 确认

结果：
同一个事件出现在两个人的人生中

--------------------------------

6）关系网络（隐含价值）

基于共同节点：
系统可以生成：
- 人与人关系强度
- 共同经历数量
- 人生交集网络图

--------------------------------

7）隐私控制（必须）

每个节点：
- private（仅自己）
- friends（指定人）
- public（公开）

--------------------------------

三、产品形态（用户看到的）

1）输入框（最核心）
一句话记录人生

2）时间线（主视图）
像滚动人生

3）地图（可选）
在哪发生

4）关系图（后期）
和谁走过人生

--------------------------------

四、技术本质（系统层）

这是一个：

结构化事件数据库 + AI解析系统 + 社交关系网络

--------------------------------

五、数据模型（核心抽象）

LifeNode（事件）
NodeParticipant（人）
User（用户）

关系：
User ←→ NodeParticipant ←→ LifeNode

--------------------------------

六、发展路径（不能乱）

阶段1（必须先做）：
单人节点系统
→ 输入 → 存储 → 时间线

阶段2：
AI解析（自动结构化）

阶段3：
多人节点（社交）

阶段4：
关系网络 / 可视化

--------------------------------

七、一句话总结

LifeGraph =
把人生拆成“节点”，
再把节点连成“网络”
